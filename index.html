the obvious objection

"Why not just keep notes in a plain file?"

A notes file works for one project, for a while — and plenty of people do exactly that (an AI assistant will happily read a file you leave for it). But nothing guards it. Any assistant that edits the file can also wipe it by accident — which is exactly what happened to one project here, until a whole-file overwrite erased its notes. Nothing catches repeats, so the same lesson piles up five times under five slightly different names. And it can't travel — a fix written in one project's file stays stuck in that project.

Brain fixes all three with one gatekeeper. A single checker program reads every new entry — is it well-formed, is it actually new, is it worth keeping — before it's allowed to join the shared logbook, and it's the only thing permitted to write there. So one person's mistake sets aside that one entry for a human to look at. It can never take down everyone else's.
